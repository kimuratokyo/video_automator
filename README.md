# 🎬 Video Automator

画像・動画素材をGUI上で順序付けし、FFmpegで高速に動画結合するWindowsデスクトップアプリケーションです。

---

## 📋 概要

| 項目 | 内容 |
|------|------|
| **目的** | 画像・動画素材を順番に並べて1本の動画に結合する |
| **処理方式** | FFmpegをバックエンドとして利用（ローカル完結） |
| **GUI** | PySide6 (Qt) によるネイティブデスクトップUI |
| **対象OS** | Windows 10 / 11 |

---

## 🖥️ 実行環境

### 必須

| ソフトウェア | バージョン | 備考 |
|------------|-----------|------|
| **Python** | 3.13以上 | [python.org](https://www.python.org/downloads/) |
| **uv** | 最新版 | [インストール方法](https://docs.astral.sh/uv/getting-started/installation/) |
| **FFmpeg** | 7.x 推奨 | 環境変数PATHに登録が必要 |

### オプション

| ソフトウェア | 用途 |
|------------|------|
| **NVIDIA GPU + ドライバー** | GPUエンコード（h264_nvenc）を使用する場合 |

### FFmpegのインストール

1. [gyan.dev](https://www.gyan.dev/ffmpeg/builds/) から `ffmpeg-release-essentials.zip` をダウンロード
2. 任意のフォルダに展開（例: `C:\ffmpeg`）
3. `C:\ffmpeg\bin` をシステム環境変数 `PATH` に追加
4. コマンドプロンプトで確認:
   ```
   ffmpeg -version
   ```

## 🚀 環境構築・起動方法

### 初回セットアップ

```bash
# 1. リポジトリをクローン or ダウンロード
cd video_automator

# 2. uv で依存関係をインストール
uv sync
```

### 起動方法

#### 方法1: バッチファイル（推奨）

`start.bat` をダブルクリックするだけで起動します。

- 初回実行時は自動で `uv sync` を実行し仮想環境を構築します
- 2回目以降は即座にアプリが起動します

#### 方法2: コマンドライン

```bash
cd video_automator
uv run python src/main.py
```

## 🎯 機能仕様

### 対応フォーマット

| 種類 | 拡張子 |
|------|--------|
| **動画** | `.mp4`, `.mov`, `.avi`, `.mkv`, `.wmv` |
| **画像** | `.jpg`, `.jpeg`, `.png`, `.bmp`, `.webp` |
| **BGM** | `.mp3`, `.wav`, `.aac`, `.flac`, `.ogg`, `.m4a` |

### 素材リスト

- **ドラッグ＆ドロップ**: ファイル/フォルダをウィンドウにD&Dして追加
- **ファイル選択**: 「＋ 追加」ボタンからファイルダイアログで追加
- **順序変更**: リスト内ドラッグ or ▲▼ボタンで並べ替え
- **削除**: 選択中のアイテムを削除 or 全消去

### トランジション設定

| トランジション | FFmpegフィルター | 説明 |
|---------------|-----------------|------|
| None | （なし） | カット切り替え |
| Fade | `fade` | フェードイン/アウト |
| Wipe | `wipeleft` | ワイプ（左方向） |
| Slide | `slideleft` | スライド（左方向） |

- **トランジション秒数**: 0.1〜5.0秒（デフォルト: 1.0秒）

### 画像設定

- **表示時間**: 1.0〜30.0秒（デフォルト: 5.0秒）
- 画像は指定秒数の動画として自動変換されます

### 音声設定

| モード | 説明 |
|--------|------|
| **素材の音声をそのまま結合** | 各動画素材の音声をそのまま結合（画像は無音） |
| **BGMのみ使用** | 素材の音声をミュートし、選択したBGMのみ使用 |
| **素材音声 + BGMをミックス** | 素材音声とBGMを重ねて出力 |

- **BGMループ**: ONにすると動画の長さに合わせてBGMを自動ループ

### エンコーダー

| エンコーダー | 種類 | 特徴 |
|-------------|------|------|
| `h264_nvenc` | GPU (NVIDIA) | 高速だがNVIDIA GPU必須 |
| `libx264` | CPU | 汎用的、どの環境でも動作 |

- NVIDIA GPUが検出されない場合、GPU選択肢は自動的に無効化されます
- 出力は H.264 High Profile / Level 4.1 / yuv420p で、Windows標準プレーヤーで再生可能です

### 出力仕様

| 項目 | 値 |
|------|-----|
| **コンテナ** | MP4 |
| **映像コーデック** | H.264 (High Profile, Level 4.1) |
| **ピクセルフォーマット** | yuv420p |
| **解像度** | 1920×1080 (Full HD) |
| **フレームレート** | 30fps |
| **音声コーデック** | AAC 192kbps |
| **MP4最適化** | faststart（ストリーミング対応） |

- 異なる解像度の素材は自動的にレターボックス/ピラーボックスで1920×1080に正規化されます

### その他

- **設定の自動保存**: トランジション、画像表示時間、音声モード、エンコーダー選択、ウィンドウ位置などの設定が終了時に自動保存され、次回起動時に復元されます
- **リアルタイムログ**: FFmpegの処理状況がログエリアにリアルタイム表示されます
- **プログレスバー**: 処理の進捗率を表示します
- **完了後フォルダオープン**: 生成完了後、出力先フォルダを自動で開くことができます

## 📁 ディレクトリ構造

```
video_automator/
├── start.bat           # ダブルクリック起動スクリプト
```

## ⚙️ 設定ファイル

設定は以下のパスにJSON形式で保存されます:

```
%USERPROFILE%\.video_automator\config.json
```

```json
{
  "encoder": "cpu",
  "image_duration": 5.0,
  "transition": "Fade",
  "transition_duration": 1.0,
  "audio_mode": "bgm_only",
  "bgm_loop": true,
  "last_output_dir": "C:/Users/user/Videos",
  "window_geometry": { "x": 100, "y": 100, "width": 1100, "height": 750 }
}
```

## 🔧 トラブルシューティング

| 問題 | 対処法 |
|------|--------|
| `FFmpegが見つかりません` | FFmpegをインストールし、PATHに `ffmpeg.exe` のあるフォルダを追加 |
| `uv が見つかりません` | [uv公式サイト](https://docs.astral.sh/uv/getting-started/installation/)からインストール |
| 動画が再生できない | 出力はH.264/AACのMP4なので、Windows標準プレーヤーで再生可能です |
| GPU選択肢が無効 | NVIDIA GPUとドライバーが必要です。CPUエンコーダーをご利用ください |
